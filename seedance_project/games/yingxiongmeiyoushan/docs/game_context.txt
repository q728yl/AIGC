# 英雄没有闪 游戏设定集 & 风格指南

> 系统提示：此文件定义《英雄没有闪》的专属内容风格。所有生成的素材规划、分镜脚本、视频 Prompt 与补充资产，都必须在遵守共享 `seedance_guide.txt` 的前提下，优先贴合本游戏“竖屏横版卷轴放置 ARPG 刷宝”的轻松刷图感、Q 萌中世纪视觉与炸屏数值爽感。

## 1. 核心视觉风格 (Art Style - Cute Medieval Loot ARPG)
《英雄没有闪》采用 Q 萌中世纪美学，整体画面兼具古拙场景、奇异魔物和卡通诙谐气质，不走传统暗黑刷宝游戏那种沉重、压抑、写实的方向。战斗特效整体不追求超复杂电影特效，而是以清晰、轻快、刷宝爽感和伤害跳字炸屏为核心。

- Style Keywords: `cute medieval fantasy`, `stylized mobile ARPG`, `vertical screen side-scrolling combat`, `cartoon monster design`, `light loot game aesthetic`, `clean combat readability`
- Visual Tone: 轻松、诙谐、爽快、刷图效率高，不阴森，不苦大仇深
- Color Palette: 中世纪棕灰石材、暖金掉落光、技能红橙紫蓝能量色、怪物夸张亮色点缀
- Negative Direction: `realistic dark fantasy`, `grimdark`, `photorealistic`, `hyper gritty`, `overly horror`, `overly futuristic sci-fi`, `low readability`

## 2. 角色原型 (Character Archetypes)
- 主角职业 (Heroes): 以战士、秘法师、暗影游侠等职业体系为核心，整体造型偏 Q 萌卡通化，但仍保留 ARPG 的职业识别度，例如重甲、法袍、枪械、暗影能量、双手武器等。
- 怪物与 Boss (Enemies): 魔物造型奇异夸张，可包含沼泽怪、亡灵、秘法造物、地牢生物、扭曲野兽等，轮廓要鲜明，便于快速识别和割草清场。
- 神宠与侍从 (Companions): 宠物、侍从、助战单位通常提供清怪、增伤、控能或辅助效果，视觉上可以围绕主角移动或在技能瞬间介入战场。
- 统一特征 (Shared Trait): 所有角色都要保持手游可读性，比例略卡通，技能辨识度高，装备和外观支持明显幻化差异。

## 3. 战斗机制与视觉反馈 (Combat & Loot Feedback)
本游戏的核心不是硬核手操，而是“自动刷图 + 核心爽点保留”的高效率战斗体验。Prompt 里必须体现清怪、爆装、伤害数字、Boss 命中反馈和掉落光柱。

- 自动战斗 (Auto Combat): 主角可持续自动攻击、自动释放技能，动作衔接流畅，不强调复杂手动连招。
- 割草清场 (Mob Clear): 群怪战斗应突出快速清场、范围技能覆盖和神宠 / 侍从辅助秒怪的爽感。
- 刷宝反馈 (Loot Drop): 击杀精英或 Boss 后，应出现明显掉落光柱、装备掉落提示或宝物感知信号，突出“刷到东西”的即时满足。
- 伤害炸屏 (Damage Spam): 构筑成型后，画面中可出现密集的伤害跳字、暴击数字、连段数字或多段结算反馈，但仍需保持主体可读。
- Boss 爽点 (Boss Impact): 打 Boss 时必须保留关键命中反馈、技能爆发点、血量压制感和阶段性掉落价值。
- Buff / Debuff: 可结合蚀梦压制、赐福、增伤、减伤、控场、穿透、持续伤害等刷图向增益或压制机制。

## 4. 职业与流派特征 (Classes & Builds)
- 暗影游侠 (Shadow Ranger): 枪械与暗影能量结合的远程 / 中距离职业，可走神射、奇袭、武器专家、枪斗术爆发、蓄力穿透等方向，强调高机动、穿透、控场和瞬时爆发。
- 战士系流派 (Warrior Builds): 如雷旋风、原力锤、雷锤等，强调近身范围打击、生存能力、暴击或高频清怪。
- 秘法师系流派 (Mage Builds): 如电矛、火眼等，强调远程持续输出、易伤叠加、燃烧、电系爆发和法术面积覆盖。
- 构筑逻辑 (Build Identity): 技能、符文、暗能套装、装备词缀、侍从和神宠共同决定流派方向；生成内容时应突出“同一职业也有不同玩法流派”的自由组合感。

## 5. 镜头语言与构图 (Camera & Composition)
- 核心视角必须服务“竖屏横版卷轴”体验，优先使用 `vertical screen side-scrolling battle layout`、`portrait mobile combat framing`、`single-hand readable composition`。
- 战场通常以竖屏为画幅，角色在横向卷轴场景中推进或遭遇敌群，因此需要兼顾纵向 UI 空间与横向战斗路径。
- 镜头不要做过度复杂的大范围运镜，更适合轻微跟随、稳定推移和清晰的战斗主体居中。
- 如果首图已经锁定角色位置、怪物方向或上下 UI 布局，后续 Prompt 必须严格沿用，不能随意改成别的阵营关系。

## 6. 场景设定 (Environments)
- 常见场景包括沼泽秘殿、幽火沼泽、秘法圣殿、地牢、废墟走廊、古神遗迹等，整体带有中古奇幻 + 轻度魔幻的副本氛围。
- 场景应服务刷图与跑图节奏，保留横版推进感、地面层次、障碍结构、秘境空间或 Boss 房间的辨识度。
- 背景可以有古拙纹理和奇幻装饰，但不能盖过角色、技能和掉落反馈。

## 7. 材质与画质要求 (Material & Quality)
- 角色、怪物、装备和技能特效都应保持手游级清晰度，轮廓明确，适合竖屏快速扫读。
- 掉落光柱、装备闪光、伤害数字、技能边缘必须醒目，优先保证“战利品反馈”和“战斗可读性”。
- 整体成片质感应更接近高质量放置刷宝手游宣传图或实机强化版，而不是主机级写实 ARPG。

## 8. Director 使用说明 (Director Notes)
- 在遵循共享 `seedance_guide.txt` 的结构、镜头与物理反馈规则时，本游戏必须优先体现“竖屏横版放置刷宝”的轻松体验，而不是高压、高操作门槛的硬核动作游戏。
- 若用户描述不充分，优先补全：主角职业与流派、怪物波次、刷图路径、Boss 爆发节点、掉落光柱、伤害数字炸屏、宠物或侍从辅助。
- 最终视频脚本应突出“自动刷图也很爽”的节奏感，表现为清怪效率高、掉落反馈强、构筑成型后数字与技能一起爆发。
- 对于强流派演出，可强调雷旋风回血续航、火锤 / 雷锤高爆发、秘法师持续电火压制、暗影游侠穿透或机动清场等职业特征。
- 避免把本游戏写成纯站桩数值游戏，也避免写成过于复杂的主机动作游戏；重点是“轻松、爽快、刷宝、有成长闭环”。
